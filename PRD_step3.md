# PRD Step 3 — 사용자 프로필 및 레시피 저장

## 개요

사용자 프로필을 생성하고, 2단계에서 생성된 레시피를 저장·조회·삭제하는 개인화 기능을 제공한다.
저장소는 `localStorage`를 사용하여 서버 없이도 동작한다.

---

## 목표

- 사용자별 식단 선호도와 알레르기 정보를 저장하여 레시피 추천 품질을 향상한다.
- 마음에 드는 레시피를 북마크하고 언제든지 다시 볼 수 있게 한다.
- 프로필 정보가 1~2단계에 자동으로 반영되는 기반을 구축한다.

---

## 사용자 스토리

| # | As a… | I want to… | So that… |
|---|-------|-----------|----------|
| 1 | 신규 사용자 | 프로필을 등록하고 싶다 | 나의 식단 조건에 맞는 레시피를 추천받을 수 있다 |
| 2 | 사용자 | 레시피를 저장하고 싶다 | 나중에 다시 확인할 수 있다 |
| 3 | 사용자 | 저장된 레시피 목록을 보고 싶다 | 어떤 레시피를 북마크했는지 한눈에 볼 수 있다 |
| 4 | 사용자 | 저장된 레시피를 삭제하고 싶다 | 불필요한 레시피를 정리할 수 있다 |
| 5 | 사용자 | 프로필을 수정하고 싶다 | 변경된 식단 정보를 반영할 수 있다 |

## 기능 요구사항

### F1. 사용자 프로필

**프로필 데이터 구조**

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | `crypto.randomUUID()` 자동 생성 |
| `name` | string | 사용자 이름 (필수) |
| `dietary_preferences` | string[] | 식단 선호 (채식, 비건, 글루텐프리, 유제품프리) |
| `allergies` | string[] | 알레르기 재료 목록 |
| `disliked_ingredients` | string[] | 기피 재료 목록 |
| `preferred_cook_time` | number | 선호 조리 시간 (분) |
| `servings` | number | 기본 인분 수 |
| `created_at` | ISO8601 | 생성 일시 |
| `updated_at` | ISO8601 | 수정 일시 |

**프로필 생성/수정 화면 필드**
- 이름 (텍스트 입력, 필수)
- 식단 선호 (체크박스: 채식 / 비건 / 글루텐프리 / 유제품프리)
- 알레르기 재료 (태그 입력)
- 기피 재료 (태그 입력)
- 기본 조리 시간 (드롭다운)
- 기본 인분 수 (숫자 입력)

**프로필 → 1~2단계 자동 적용**
- 2단계 진입 시 `allergies`, `disliked_ingredients`, `preferred_cook_time`, `servings`를 옵션 기본값으로 자동 채움
- 1단계 API 호출 시 알레르기 재료를 제외하도록 프롬프트에 반영

### F2. 레시피 저장

**저장 데이터 구조**
```json
{
  "id": "saved_001",
  "recipe": {
    "name": "달걀 채소볶음",
    "description": "...",
    "cook_time_minutes": 20,
    "servings": 2,
    "difficulty": "쉬움",
    "ingredients": [...],
    "steps": [...],
    "tips": "...",
    "nutrition": { "calories": 320, "protein": "18g", "carbs": "12g", "fat": "22g" }
  },
  "source_ingredients": ["달걀", "당근", "양파"],
  "memo": "다음엔 간장 줄이기",
  "saved_at": "2024-01-15T12:00:00Z"
}
```

**저장 동작**
- 2단계 레시피 상세 화면의 "저장하기" 버튼 클릭
- 동일 레시피명이 이미 저장된 경우 중복 저장 방지 안내
- 저장 시 사용자 메모 추가 가능

### F3. 저장된 레시피 목록

- 카드 그리드 UI로 전체 저장 목록 표시
- 정렬: 저장일 최신순 / 조리 시간 짧은순 / 이름 가나다순
- 검색: 레시피명으로 필터

### F4. 레시피 삭제

- 각 카드의 삭제 버튼 클릭 → 확인 후 삭제
- 삭제 후 목록 즉시 갱신

---

## 비기능 요구사항

| 항목 | 요건 |
|------|------|
| 저장소 | `localStorage` (추가 서버/DB 불필요) |
| 최대 저장 레시피 수 | 100개 (초과 시 저장 전 안내) |
| 오프라인 지원 | 네트워크 없이도 저장 목록 조회 가능 |
| 사용자 식별 | `crypto.randomUUID()` — 별도 로그인 없음 |

## 기술 스택

| 영역 | 기술 |
|------|------|
| 저장소 | `localStorage` (브라우저 내 JSON) |
| 사용자 식별 | `crypto.randomUUID()` (로그인 없음) |
| 상태 공유 | `sessionStorage`로 1~2단계에 프로필 자동 적용 |

## 데이터 흐름

```
[프로필 설정]
  dietary, allergies, disliked, cook_time, servings
        │
        ▼
  localStorage["user_profile"]
        │
        ├──→ [Step 1 진입 시] 알레르기 재료 프롬프트에 주입
        └──→ [Step 2 진입 시] 옵션 기본값 자동 채움
                                      │
                                      ▼
                            [레시피 생성 결과]
                                      │
                                [저장하기 클릭]
                                      │
                                      ▼
                        localStorage["saved_recipes"]
                                      │
                                      ▼
                            [저장된 레시피 목록]
```

---

## 1~3단계 통합 흐름

```
[Step 1] 냉장고 사진 업로드
         → 재료 인식 (google/gemma-4-31b-it:free)
         → 재료 목록 편집
              │
              ▼
[Step 2] 옵션 선택 + 레시피 생성 (google/gemma-4-26b-a4b-it:free)
         → 레시피 3개 카드 표시
         → 상세 보기
              │
              ▼ [저장하기]
[Step 3] localStorage에 레시피 저장
         → 저장 완료 메시지
         → 보관함에서 언제든 재확인
```
